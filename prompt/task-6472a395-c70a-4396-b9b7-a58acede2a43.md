# توابع بزرگ و تکراری برای فراخوانی Gemini API در backend/server.js

## Raw Idea

فایل backend/server.js با بیش از 1400 خط، حاوی چندین تابع است که همگی فراخوانی‌های مشابهی به Gemini API انجام می‌دهند. توابع analyzeWithGemini (خط 273)، analyzeWithGeminiFileAPIWithModel (خط 395)، و endpointهای /api/gemini/chat (خط 56) و /api/gemini/tts (خط 117) همگی URL ثابت، هدرهای مشابه و پردازش پاسخ یکسان دارند. این duplication باعث افزایش حجم فایل، دشواری در نگهداری (مثلاً تغییر مدل یا API key نیاز به تغییر در چندین مکان دارد) و افزایش احتمال خطا می‌شود. همچنین تابع uploadToGeminiFileAPI (خط 297) با بیش از 90 خط، بسیار طولانی است و منطق آپلود resumable را در خود دارد که می‌تواند به یک سرویس جدا منتقل شود.
