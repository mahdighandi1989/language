# افزودن تست واحد برای endpointهای اصلی API

## Raw Idea

پروژه فاقد هرگونه تست واحد یا integration test است. با توجه به پیچیدگی backend (WebSocket، آپلود فایل، فراخوانی‌های خارجی Gemini API، TTS) و وجود بیش از 10 endpoint، عدم وجود تست باعث می‌شود که تغییرات در کد به راحتی باعث بروز regression bugs شود. به عنوان مثال، endpoint /api/gemini/chat (خط 56) منطق پیچیده‌ای برای پاکسازی payload و مدیریت includeAudio دارد که بدون تست قابل اطمینان نیست. همچنین WebSocket server (خط 45) و File Analysis API (خطوط 242+) نیاز به تست دارند.
