# Secure backend API endpoints

## Raw Idea

The backend server.js exposes multiple API endpoints (/api/gemini/chat, /api/gemini/tts, /api/list-models, /api/test-gemini, /api/health, and file analysis endpoints) without any authentication or authorization. Any client that can reach the server can call these endpoints. This is particularly dangerous because the /api/gemini/chat and /api/gemini/tts endpoints use the GEMINI_API_KEY to make requests to Google's Gemini API, potentially allowing an attacker to consume the API quota or incur costs. The /api/list-models and /api/test-gemini endpoints also expose the API key prefix in responses, which aids attackers in identifying the key.

```
app.use(cors({ origin: process.env.ALLOWED_ORIGINS?.split(',') || 'http://localhost:5173' }));

// Authentication middleware
const authenticateRequest = (req, res, next) => {
  const apiKey = req.headers['x-api-key'];
  if (!apiKey || apiKey !== process.env.APP_API_KEY) {
    return res.status(401).json({ error: 'Unauthorized' });
  }
  next();
};
app.use('/api', authenticateRequest);
```
