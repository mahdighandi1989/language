# راهنمای ربات تلگرام (دو طرفه)

این راهنما توضیح می‌دهد چطور ربات تلگرام برنامهٔ آموزش لهجهٔ لبنانی را راه‌اندازی
و استفاده کنید. ربات سه کار اصلی انجام می‌دهد:

1. **نوتیفیکیشن** — رویدادهای مهم سیستم (ثبت‌نام، خطا، آپلود، پایان تمرین) را به
   تلگرام شما می‌فرستد.
2. **کنترل از راه دور** — دیدن، افزودن، ویرایش و حذف دروس و محتوا، و آپلود فایل،
   همه از داخل تلگرام.
3. **تمرین با استاد هوش مصنوعی** — انتخاب موضوع، گفتگوی متنی، و پیام صوتی (میکروفون)
   که رونویسی و تحلیل می‌شود؛ پاسخ استاد به‌صورت متن **و صدا** در تلگرام می‌آید.

> اگر متغیر `TELEGRAM_BOT_TOKEN` تنظیم نشده باشد، ربات غیرفعال می‌ماند و بقیهٔ
> برنامه عادی کار می‌کند. هیچ راه‌اندازی اجباری نیست.

---

## ۱. راه‌اندازی (یک‌بار)

### ساخت ربات
1. در تلگرام به [@BotFather](https://t.me/BotFather) پیام بدهید و `/newbot` را
   بفرستید.
2. یک نام و یک نام‌کاربری (که به `bot` ختم می‌شود) انتخاب کنید.
3. توکنی که BotFather می‌دهد را کپی کنید.

### تنظیم متغیرهای محیطی
در فایل `backend/.env` (یا داشبورد سرویس میزبان، طبق `render.yaml`) مقادیر زیر را
بگذارید:

| متغیر | توضیح |
| --- | --- |
| `TELEGRAM_BOT_TOKEN` | توکن گرفته‌شده از BotFather. اگر خالی باشد به `NOTIFY_TELEGRAM_BOT_TOKEN` برمی‌گردد. |
| `TELEGRAM_MODE` | `polling` (پیش‌فرض) یا `webhook`. |
| `TELEGRAM_WEBHOOK_URL` | برای حالت webhook؛ آدرس عمومی HTTPS که به `/api/telegram/webhook` ختم می‌شود. |
| `TELEGRAM_WEBHOOK_SECRET` | رشتهٔ مخفی برای جلوگیری از درخواست‌های جعلی به webhook (توصیه‌شده). |
| `TELEGRAM_ALLOWED_USER_IDS` | شناسه‌های عددی کاربرانی که اجازهٔ دستورات نوشتاری (افزودن/ویرایش/حذف/آپلود) دارند. خالی = همه. |
| `TELEGRAM_ADMIN_IDS` | شناسه‌های عددی ادمین‌ها (دسترسی به `/admin_logs`). |
| `TELEGRAM_CHAT_ID` | چت پیش‌فرض برای نوتیفیکیشن‌های سیستمی وقتی کاربری حساب وصل نکرده. |
| `TELEGRAM_DATA_DIR` | پوشهٔ ذخیرهٔ JSON ربات (پیش‌فرض: `backend/data`). |

> شناسهٔ عددی تلگرام خود را از رباتی مثل [@userinfobot](https://t.me/userinfobot)
> بگیرید.

### حالت polling در مقابل webhook
- **polling** (پیش‌فرض): نیازی به آدرس عمومی نیست؛ سرور خودش به‌صورت دوره‌ای از
  تلگرام به‌روزرسانی می‌گیرد. برای توسعه و اجرای ساده مناسب است.
- **webhook**: تلگرام به‌روزرسانی‌ها را به آدرس HTTPS شما POST می‌کند. وقتی
  `TELEGRAM_WEBHOOK_URL` تنظیم شود به‌صورت خودکار انتخاب می‌شود. حتماً
  `TELEGRAM_WEBHOOK_SECRET` را هم بگذارید تا درخواست‌های جعلی رد شوند.

## ۲. اتصال حساب (Account Linking)

برای اینکه نوتیفیکیشن‌ها و تمرین‌های شما به حساب کاربری برنامه وصل شود:

1. در تلگرام به ربات `/login` بفرستید. ربات یک **کد اتصال** کوتاه می‌دهد.
2. در برنامهٔ وب وارد حساب خود شوید و کد را در بخش اتصال تلگرام وارد کنید
   (`POST /api/telegram/link` با احراز هویت).
3. پس از تأیید، چت تلگرام شما به حساب‌تان متصل می‌شود.

برای قطع اتصال `/logout` را بفرستید.

> `/verify` هم‌معنی `/login` است.

---

## ۳. دستورها

دستورها را می‌توانید مستقیم تایپ کنید یا از `/help` لیست کامل را ببینید. دسترسی
هر دستور یکی از این سه سطح است: **عمومی** (همه)، **نوشتاری** (لیست مجاز)،
**مدیریت** (ادمین).

### عمومی
| دستور | کار |
| --- | --- |
| `/start` | شروع و معرفی ربات |
| `/help` | نمایش راهنمای دستورها |
| `/status` | وضعیت سیستم |
| `/cancel` | لغو عملیات جاری (مثلاً وسط افزودن درس) |

### حساب کاربری
| دستور | کار |
| --- | --- |
| `/login` | دریافت کد اتصال حساب |
| `/verify` | هم‌معنی `/login` |
| `/logout` | قطع اتصال حساب |

### دروس
| دستور | کار |
| --- | --- |
| `/courses` | لیست دروس |
| `/course_content <id>` | محتوای یک درس |
| `/add_course` | افزودن درس جدید (گفتگوی مرحله‌به‌مرحله) — نوشتاری |
| `/edit_course <id>` | ویرایش درس — نوشتاری |
| `/delete_course <id>` | حذف درس و محتوای آن — نوشتاری |
| `/upload <course_id>` | آپلود محتوا؛ بعد از دستور فایل را بفرستید — نوشتاری |
| `/delete_content <course_id> <content_id>` | حذف یک محتوا — نوشتاری |

### تمرین با استاد هوش مصنوعی
| دستور | کار |
| --- | --- |
| `/practice` | شروع/اتصال به تمرین با استاد |
| `/practice_topic <session_id>` | انتخاب موضوع (از دکمه‌های شیشه‌ای) |
| `/practice_chat <session_id>` | شروع گفتگو؛ بعد از آن هر پیام متنی یا صوتی به استاد می‌رود |
| `/practice_end <session_id>` | پایان جلسه و دریافت خلاصه |
| `/practice_history` | تاریخچهٔ جلسات |
| `/practice_details <session_id>` | جزئیات و پیام‌های یک جلسه |

### تنظیمات و مدیریت
| دستور | کار |
| --- | --- |
| `/settings` | روشن/خاموش کردن انواع نوتیفیکیشن و تنظیمات ربات |
| `/admin_logs` | آخرین لاگ‌های ربات — فقط ادمین |

## ۴. کار با محتوا (افزودن/ویرایش/آپلود)

### افزودن درس
1. `/add_course` را بفرستید.
2. ربات قدم‌به‌قدم عنوان و توضیح درس را می‌پرسد؛ هر بار یک پیام پاسخ بدهید.
3. در پایان درس ساخته می‌شود و شناسهٔ آن نمایش داده می‌شود.
4. برای انصراف هر لحظه `/cancel` بزنید.

### آپلود محتوا
1. `/upload <course_id>` را بفرستید.
2. سپس یک فایل (PDF، صوت، …) را در همان چت بفرستید.
3. فایل به آن درس اضافه می‌شود.

### حذف
- `/delete_content <course_id> <content_id>` یک محتوا را حذف می‌کند.
- `/delete_course <id>` کل درس و محتوای آن را حذف می‌کند (با تأیید دکمه‌ای).

---

## ۵. جریان تمرین با استاد (متن + صدا)

1. `/practice` → ربات یک جلسهٔ جدید می‌سازد و `session_id` می‌دهد.
2. `/practice_topic <session_id>` → از دکمه‌های شیشه‌ای یکی از موضوع‌ها را بزنید
   (مثلاً سلام و احوال‌پرسی، خرید، رستوران…).
3. `/practice_chat <session_id>` → از این لحظه گفتگو شروع است:
   - **پیام متنی** بفرستید → استاد متنی پاسخ می‌دهد.
   - **پیام صوتی** (با میکروفون تلگرام) بفرستید → صدا **رونویسی و تحلیل** می‌شود،
     و پاسخ استاد هم به‌صورت **متن و هم پیام صوتی** برمی‌گردد.
4. `/practice_end <session_id>` → جلسه بسته می‌شود و خلاصه‌ای از تمرین می‌گیرید.
5. بعداً با `/practice_history` و `/practice_details <session_id>` می‌توانید
   جلسه‌ها را مرور کنید.

---

## ۶. نوتیفیکیشن‌ها

ربات این رویدادها را اطلاع می‌دهد (قابل روشن/خاموش از `/settings`):
- **ثبت‌نام** کاربر جدید
- **خطاهای** مهم سیستم
- **آپلود** محتوا
- **تمرین** (شروع/پایان جلسه)

اگر کاربری حساب وصل نکرده باشد، نوتیفیکیشن‌های سیستمی به `TELEGRAM_CHAT_ID`
(یا `NOTIFY_TELEGRAM_CHAT_ID`) می‌رود.

---

## ۷. عیب‌یابی

| مشکل | علت احتمالی / راه‌حل |
| --- | --- |
| ربات پاسخ نمی‌دهد | `TELEGRAM_BOT_TOKEN` تنظیم نشده یا اشتباه است. سرور را با توکن درست دوباره اجرا کنید. |
| دستورهای نوشتاری کار نمی‌کنند | شناسهٔ شما در `TELEGRAM_ALLOWED_USER_IDS` نیست. شناسه را اضافه کنید (یا لیست را خالی بگذارید تا همه مجاز شوند). |
| `/admin_logs` می‌گوید دسترسی ندارید | شناسهٔ شما در `TELEGRAM_ADMIN_IDS` نیست. |
| webhook کار نمی‌کند | آدرس باید HTTPS عمومی و به `/api/telegram/webhook` ختم شود؛ `TELEGRAM_WEBHOOK_SECRET` باید با مقدار سرور یکی باشد. |
| `/api/telegram/status` می‌گوید disabled | توکن تنظیم نشده؛ ربات در حالت no-op است. |
| نوتیفیکیشن نمی‌آید | حساب را با `/login` وصل کنید یا `TELEGRAM_CHAT_ID` را تنظیم کنید؛ نوع نوتیفیکیشن را در `/settings` بررسی کنید. |

---

## ۸. نکات امنیتی

- توکن ربات و کلیدها هرگز در لاگ‌ها نمایش داده نمی‌شوند (به‌صورت خودکار پنهان
  می‌شوند).
- در حالت webhook حتماً `TELEGRAM_WEBHOOK_SECRET` را تنظیم کنید؛ درخواست‌های بدون
  هدر درست با خطای ۴۰۳ رد می‌شوند.
- مسیر webhook از محدودکنندهٔ نرخ عمومی API عبور داده می‌شود تا تلگرام مسدود نشود،
  اما با راز هدر محافظت می‌شود.
- دستورهای نوشتاری و مدیریتی پشت لیست‌های مجاز هستند؛ آن‌ها را محدود نگه دارید.
